# phase7-and-docs — SPEC

## 승인 전 확인
- 외부연동 패키지(database·search·storage·외부 벡터 백엔드·MCP HTTP 전송)의 완료를, 라이브 외부 서비스 없이
  빌드와 스텁·단위 테스트로 관찰하고 실제 서비스 대상 통합 검증은 크리덴셜이 있는 환경에서만 수행(없으면 skip)하는
  것으로 두는 것이 맞는가. 기존 레포 관례(Ollama 임베딩·a2a·mcp stdio 테스트가 크리덴셜/서버 부재 시 skip)와
  정합하는 기준이다. 관련 본문: §3, §5
- README 동기화 범위를, 감사에서 식별된 드리프트 전체(임베딩 프로바이더, 제외된 시각화·로컬 벡터 백엔드,
  multiagent 계층 서술과 §27 자기모순, 이번까지 제거된 필드)로 두는 것이 맞는가. 관련 본문: §5

## 1. 범위
두 갈래의 남은 작업을 함께 다룬다.

- Phase 7 외부연동 라이브러리 구현(README §16·§18~§21·§24·§26 Phase 7 중 미구현분): 관계형/벡터 DB 접근
  (`database`), 웹 검색(`search`), 외부 파일 스토리지(`storage`), `config`의 서버·에이전트 어셈블리 함수,
  `vectorstore`의 외부(pgvector) 백엔드, `mcp`의 HTTP 스트리밍 전송.
- 루트 명세 문서(README) 동기화: 마스터 초안이 실제 구현·phase 결정과 어긋난 지점을 실제 상태에 맞게 정정.

이미 별도 기획 문서를 가진 `trace` 구현과, README가 다운스트림 몫으로 규정한 응용 계층(`rag`/`orchestrator`)은
이 범위에 넣지 않는다(§4).

## 2. 목표
라이브러리 명세(README §1)가 목표로 한 외부연동 계층을 채워, 다운스트림이 웹검색·외부 벡터/관계형 DB·파일
스토리지·원격 에이전트 조립·MCP HTTP 서버까지 이 primitive 위에서 조립할 수 있게 한다. 동시에 명세 문서가 실제
구현과 어긋나 신뢰할 수 없게 된 상태를 해소해, README만 보고도 실제 동작을 정확히 파악할 수 있게 한다.

## 3. 제약
- 기존 공개 API 시그니처 호환을 유지한다. 신규 패키지 추가와 기존 패키지의 additive 확장은 허용하되, 이미
  노출된 함수·타입의 형태는 바꾸지 않는다.
- README §28-1의 패키지 경계 규칙(순환 import 차단, leaf 유지, 좁은 인터페이스 주입)을 지킨다. 특히 `config`는
  leaf로 유지하고, 어셈블리 함수는 상위 패키지 구체 타입을 직접 반환하지 않는다.
- 외부 SDK·의존성은 README §28이 지목한 생태계(Supabase/pgvector, Tavily 계열 검색, Google Drive API,
  MCP Go SDK)를 사용한다.
- 라이브 외부 서비스가 없는 환경에서도 `go build ./...`와 단위 테스트가 통과해야 한다. 실제 서비스 대상 통합
  검증은 크리덴셜이 있을 때만 수행하고, 없으면 skip한다.
- 기존 테스트가 계속 통과해야 한다.

## 4. 제외 범위
- `trace` 구현. 이미 spec·analysis·implement 문서를 가지므로 별도 진행한다.
- 응용 계층 `rag`·`orchestrator`의 전용 패키지화. README가 다운스트림 직접 구현 대상으로 규정했다.
- `vectorstore`의 Chroma 로컬 영속 백엔드. 현재 의존이 경계 테스트로 차단돼 있고 README §26상 Phase 3 항목이라
  이 feature의 Phase 7 범위 밖이다.
- OpenAI 외 챗 프로바이더(gemini 등)와 임베딩 프로바이더 교체.
- README 이외 문서(features/ 하위 과거 기획 문서 등)의 소급 정정.

## 5. 완료 조건
1. 호출자가 `database` 클라이언트로 관계형/벡터 DB(문서·웹콘텐츠)에 대해 삽입·전문검색·유사도검색·필터질의를
   수행할 수 있고, 도구로도 노출된다. 크리덴셜이 없는 환경에서는 관련 통합 테스트가 skip되며 빌드·단위 테스트는
   통과한다.
2. 호출자가 `search` 클라이언트로 웹 검색을 수행하고 결과를 받으며, 검색·웹콘텐츠 로딩이 도구로 노출된다.
   크리덴셜 없는 환경에서 통합 테스트는 skip, 빌드·단위 테스트는 통과한다.
3. 호출자가 `storage` 클라이언트로 외부 파일 스토리지에 업로드·다운로드·목록·폴더 조작을 수행하고 `storage_ref`로
   파일을 지칭할 수 있으며, 파일 조작이 도구로 노출된다. 크리덴셜 없는 환경에서 통합 테스트는 skip, 빌드·단위
   테스트는 통과한다.
4. `config`가 MCP 서버 설정과 에이전트 엔드포인트 설정을 조립해 반환하는 함수를 제공하며, 이때 `config`는 상위
   패키지를 import하지 않는 leaf로 유지된다(중립 타입 반환).
5. `vectorstore`의 외부(pgvector) 백엔드가 `database` 클라이언트에 위임해 유사도 검색을 제공하며, 인메모리 백엔드와
   동일한 retriever 계약으로 사용된다. 크리덴셜 없는 환경에서 통합 테스트는 skip, 빌드·단위 테스트는 통과한다.
6. `mcp`가 HTTP 스트리밍 전송으로 클라이언트 연결과 서버 노출을 지원한다. 전송 종류에 HTTP가 포함되고, 서버가
   HTTP로 도구·프롬프트를 제공할 수 있다.
7. README가 실제 구현 상태와 정합한다 — 감사에서 식별된 드리프트(임베딩 프로바이더, 의도적으로 제외된 시각화·
   로컬 벡터 백엔드, multiagent의 계층 서술과 자기모순, 이번까지 제거된 필드)가 실제 상태를 반영하도록 정정되고,
   위 §5.1~§5.6으로 새로 구현된 항목이 명세에 부합한다.
