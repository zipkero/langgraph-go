# code-drift-fixes — SPEC

## 승인 전 확인
- 제거 대상 공개 요소(챗 요청의 응답 포맷 지정 필드·상수, 에이전트 모델 요청의 미사용 메시지 필드, 체크포인트의
  부모 설정 필드)는 레포 내부 기준으로는 어디서도 참조·기록되지 않는 죽은 요소다. 다만 공개 표면이라 이 레포에
  보이지 않는 외부 다운스트림이 참조할 여지가 있다. "레포 내부 미참조"를 제거 근거로 삼는 것이 맞는가. 관련 본문:
  §3, §4

## 1. 범위
전체 코드 감사에서 드러난 결함·드리프트 중, **살아있는 공개 API의 동작과 시그니처를 바꾸지 않고 교정 가능한**
코드 정합 문제를 바로잡는다. 대상은 세 가지다.

- `vectorstore`의 색인 생성 함수가 전달받은 옵션을, 적용 후 즉시 버려지는 임시 인스턴스에 적용해 실제 스토어
  생성에 반영하지 못하는 잠재 버그.
- `llm` 챗 요청 타입에 있으나 챗 호출 경로에서 전혀 처리되지 않는 응답 포맷 지정 필드·상수(죽은 필드).
- 에이전트 모델 요청 타입과 체크포인트 타입에 선언만 되어 있고 어디서도 읽거나 채우지 않는 미사용 필드.

## 2. 목표
라이브러리 코드가 아무 효과 없는 죽은 요소를 노출하거나, 자기 설계 의도와 어긋난 동작(옵션 미반영)을 감춘
상태를 없앤다. 관찰 가능한 동작을 새로 바꾸지 않으면서, 잠재 버그가 실제 결함으로 드러나기 전에 제거하고 공개
표면의 신뢰성과 유지보수성을 높인다.

## 3. 제약
- 살아있는(레포 안에서 참조되는) 공개 API의 동작과 함수 시그니처는 바꾸지 않는다. 제거를 허용하는 대상은 레포
  안 어디서도 참조·기록되지 않는 죽은 공개 필드·상수에 한한다.
- 이번 교정은 기능 추가가 아니라 정합·버그 교정이다. 새로운 관찰 가능 동작을 도입하지 않는다.
- `go build ./...`와 기존 테스트가 계속 통과해야 한다.

## 4. 제외 범위
- 살아있는 공개 API 시그니처 변경. 특히 `checkpoint`의 스레드 식별자 추출 함수 반환형 변경, 상태 로드/저장을
  패키지 레벨 헬퍼로 신설하는 것은 하지 않는다(공개 API 호환 유지 결정).
- 응답 포맷 지정 기능의 신규 구현(챗 호출에서 json_object/text를 실제로 처리하도록 만드는 것). 이번에는 죽은
  필드 제거를 택했으므로 기능 구현은 범위 밖이다.
- Phase 7 미완 신규 구현: `database`·`search`·`storage` 패키지, `trace` 구현, `config` 어셈블리 함수,
  `vectorstore`의 외부(pgvector) 백엔드, `mcp` HTTP 전송.
- 루트 명세 초안(README)과 실제 구현·phase 결정 사이의 문서 동기화.

## 5. 완료 조건
1. 색인 생성 함수에 전달된 스토어 옵션이, 적용 즉시 버려지는 임시 인스턴스가 아니라 실제 스토어 생성에 쓰이는
   옵션 인스턴스에 적용된다. 테스트용 옵션을 전달하면 그 옵션이 호출되어 생성 경로에 반영됨이 테스트로 관찰된다.
2. 챗 요청 타입에서 챗 호출 경로가 전혀 처리하지 않던 응답 포맷 지정 필드와 그 관련 상수가 제거된다. 제거 후
   `go build ./...`와 기존 테스트가 통과하며, 구조화/JSON 출력은 종전의 전용 경로로만 계속 제공된다.
3. 에이전트 모델 요청 타입과 체크포인트 타입에서 어디서도 읽거나 채우지 않던 미사용 필드가 제거된다. 제거 후
   `go build ./...`와 기존 테스트가 통과한다.
