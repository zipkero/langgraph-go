# code-drift-fixes — 구현 체크리스트

`features/20260701-001-code-drift-fixes/spec.md` §5와 `analysis.md`로부터 도출한 순수 실행 체크리스트다. 설계
근거는 analysis.md에 있다. spec §5의 세 완료 조건이 서로 독립이고 순서 제약이 없으므로(ANALYSIS §5 D3), 평면
리스트로 Task 3개를 두되 각 Task를 완료 조건 하나에 1:1 대응시켜 항목별 verify가 가능하게 한다. 공통 검증은 모든
Task에서 `go build ./...` 성공과 기존 테스트 전량 통과다.

- [x] task-001: 벡터스토어 색인 생성 옵션 반영
  - 목적: 색인 생성 함수가 전달받은 옵션을 적용 즉시 버려지는 임시 인스턴스가 아니라 실제 스토어 생성에 쓰이는
    단일 옵션 인스턴스에 적용하고, 그 인스턴스가 생성 경로로 전달되게 한다. 새 옵션 필드나 관찰 가능한 공개
    동작은 추가하지 않는다.
  - 접근: 단일 옵션 인스턴스를 만들어 전달된 옵션들을 그 인스턴스에 누적 적용하고, 같은 인스턴스를 스토어 생성
    경로로 넘긴다. 옵션 호출·반영을 관찰하는 회귀 테스트를 `package vectorstore`(내부 테스트)로 신설한다.
  - 검증 조건:
    - 결과: 옵션이 단일 인스턴스에 적용되어 생성 경로로 전달되고, 공개 시그니처·반환형은 불변이다.
    - 확인: `go build ./...` 성공 및 기존 테스트 통과. 추가로 신규 내부 테스트가 전달된 옵션의 호출·반영을
      관찰해 통과한다.
  - 참조: SPEC §5.1 / ANALYSIS §1(1), §2, §5 D1

- [x] task-002: 챗 요청의 죽은 응답 포맷 요소 제거
  - 목적: 챗 호출 경로가 전혀 처리하지 않는 응답 포맷 지정 요소를 전부 제거한다. 대상은 챗 요청 타입의 응답
    포맷 필드, 응답 포맷 타입, 응답 포맷 종류 타입, 관련 상수 3종이다. 구조화/JSON 출력은 종전의 전용 경로로만
    계속 제공한다.
  - 접근: 죽은 필드·타입·상수 선언을 제거하고, 구조화/JSON 출력을 담당하는 기존 전용 경로는 손대지 않는다.
  - 검증 조건:
    - 결과: 응답 포맷 필드·타입·종류 타입·상수 3종이 선언에서 사라지고, 구조화/JSON 출력 계약은 전용 경로로
      유지된다.
    - 확인: `go build ./...` 성공 및 기존 테스트 통과. 제거 전 챗 요청 관련 테스트가 해당 필드를 실제로
      구성하는지 점검하고, 구성한다면 함께 정리한 뒤 제거한다.
  - 참조: SPEC §5.2 / ANALYSIS §1(2), §4, §5 D2

- [x] task-003: 미사용 필드 제거(모델 요청·체크포인트)
  - 목적: 어디서도 읽거나 채우지 않는 미사용 필드를 제거한다. 대상은 에이전트 모델 요청 타입의 메시지 필드와
    체크포인트 타입의 부모 설정 필드다. 체크포인트가 계속 사용하는 설정 패키지 import는 유지한다.
  - 접근: 두 미사용 필드 선언만 제거하고, 각 패키지의 나머지 필드·메서드·import는 그대로 둔다.
  - 검증 조건:
    - 결과: 두 미사용 필드가 선언에서 사라지고, 체크포인트의 설정 import는 유지되며 나머지 공개 표면은 불변이다.
    - 확인: `go build ./...` 성공 및 기존 테스트 통과. 관련 테스트가 두 필드를 구성하지 않음을 확인한다.
  - 참조: SPEC §5.3 / ANALYSIS §1(3), §4
